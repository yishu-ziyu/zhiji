# Efficiency Agent Development Plan

## Product goal

Build the clearest possible demo for the one-person company efficiency track.

The demo should show one strong user story instead of many weak features.

## Must have

1. The page must clearly say this is the selected hackathon direction.
2. A judge must be able to run the demo with one click.
3. Meeting action items must become Kanban tasks.
4. The board must make unfinished work visible.

## Should have

1. A delivery reminder area for upcoming and blocked tasks.
2. A simple before and after message for the pitch.

## Won't do before the core demo is stable

1. Real calendar integration.
2. Real account system.
3. Real team invitation.
4. Complex multi-agent orchestration.

## Next implementation wave

1. Add a visible selected-direction banner on the efficiency page.
2. Add a demo transcript button.
3. Add a delivery reminder panel from current tasks.
4. Run E2E and build checks.
