# G2-015 验收细则（BDD）

**合同：** `docs/product/dev-contract-015-first-user-real-entry-v1.md`  
**角色：** G2 Spec · 只补验收 · 不写业务码  
**范围边界：** 不验收「整夹递归拖入 / 自动拆多项目」（合同非目标）。

## 前置（G4 空环境）

- 使用**空 data 目录**（或等价干净用户态）：无用户项目、无用户材料。
- 默认配置：**不**开演示 seed；若 `SEED_DEMO=1` 另案，见用例 E。

---

## 验收用例

### A. 首次进入诚实为空

**当我**在空 data 目录下首次打开工作台，  
**应见**无用户项目、无用户材料的诚实空态（可提示「新建项目」），  
**不应见**预置项目/卡片/任务被呈现为「我的进行中工作」或「已有局面」。

**通过线：** 空态文案或结构明确；无冒充用户工作的默认项目名/任务列表。  
**失败线：** 出现 `DEFAULT_PROJECT` / seed 项目当作用户跟进对象。

### B. 显式新建项目后才有工作台

**当我**点击新建项目、填写必填名称并确认，  
**应见**仅出现该用户自建项目，并可进入其工作台。

**当我**尝试不填名称提交，  
**应见**校验失败、项目未创建、列表无新增项。

**通过线：** 名称必填；创建后列表/导航可见该项目 ID 对应入口。  
**失败线：** 未操作即已有可进工作台；或空名可建成。

### C. 本地文件归属项目并可回看

**当我**在某用户自建项目内添加至少 1 个真实本地文件，  
**应见**该材料归属该项目 ID；在该项目内可检索到或列表可见，并可点开/预览（合同要求的「搜到/点开」至少满足其一完整路径，推荐两者）。

**当我**切换到另一项目（若仅一项目则跳过），  
**应见**该文件不出现在其他项目材料中。

**通过线：** 文件与 `projectId` 绑定；项目内可见可开。  
**失败线：** 文件进全局杂堆无项目归属；或项目内找不到刚加文件。

### D. 刷新/重启仍在；seed 不冒充

**当我**完成 B+C 后刷新页面或重启应用再进入，  
**应见**自建项目与自加材料仍在，且仍属同一项目。

**当我**在默认配置（seed 关）下重复 A，  
**应见**仍不出现未标注的预置「用户工作局面」。

**通过线：** 用户数据持久化；默认路径无 seed 冒充。  
**失败线：** 刷新后项目/材料消失；或重启后 seed 顶替用户数据展示。

### E. 演示 seed 可辨（若实现提供）

**当我**以默认配置启动（无 `SEED_DEMO=1`），  
**应见**不注入未标注示例数据，或 seed 与用户数据严格隔离且 UI 标明「示例数据」。

**当我**以 `SEED_DEMO=1`（或文档约定开关）启动，  
**应见**示例数据带明确「示例/演示」标注，不得与用户自建项目混称为同一「我的工作」。

**通过线：** 默认关；开则标注清晰。  
**失败线：** 默认即灌 seed 且无标注。

### F. 本版明确不验（防扩 scope）

以下**不作为 015 通过条件**（出现也不要求本版实现）：

- 整夹递归拖入
- 自动识别并拆成多个项目
- 完美向量库/语义检索质量

---

## G4 最小通关路径（推荐顺序）

1. 空 data → 打开 → **A**  
2. 新建项目（有名）→ **B**  
3. 添加 ≥1 真实文件 → **C**  
4. 刷新或重启 → **D**  
5. （可选）核对默认 seed 行为 → **E**

**总判定：** A–D 全过 = 015 用户行为验收通过；E 在提供 seed 开关时必过，未提供 seed 能力则 E 记 N/A 且不得默认灌假数据。
