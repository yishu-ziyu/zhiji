# Scope challenge - research cycle

## Candidate requirements

| Requirement | Owner | Keep / Cut / Defer | Reason |
|---|---|---|---|
| Prove the problem, user, and scene | User | Keep | Required for scene value |
| Prove a plausible buyer and repeatable commercial path | User | Keep | Required for commercial potential |
| Compare the old workflow and current AI/Agent products | User | Keep | Required before any innovation claim |
| Research low-friction input channels, including WeChat constraints | User | Keep research only | Input friction can invalidate the scene |
| Change application code | Engineering | Defer | Upstream product decision is unresolved |
| Create branding or a product name | Prior project | Cut | Does not answer the first three judging dimensions |
| Build a complete OPC operating system | Prior project | Cut | Too broad to validate or demonstrate causally |
| Claim automatic priority decisions | Prior discussion | Cut | Required business inputs and user trust are unproven |

## Must ship this research cycle

- A cited scene-value evidence dossier with counter-evidence.
- A commercial path that distinguishes service, productized service, SaaS product, and repeatable solution.
- A current competitor and workflow baseline showing what is and is not new.
- A single do / do-not-do / change-the-question decision for user review.

## Explicit non-goals

- No business code, UI, naming, pricing claim, market-size claim, or pitch script.
- No claim that the current commitment or Waiting-for concepts are already validated.
