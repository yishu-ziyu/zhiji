# 项目画布调查差异与处理

**日期：** 2026-07-15
**主调查稿：** `plan/spec.md`
**独立调查稿：** `plan/peer-spec.md`
**说明：** 独立调查会话超时，`peer-spec.md` 按流程使用三个独立源码调查结果降级重建；以下结论均回到当前仓库代码和产品问题核对。

## 结论

两份调查对总体结构没有冲突：项目、当前中心、一层关系、右侧说明、时间线、真实事件写回。独立调查补出了主稿遗漏的“离开时状态”和“原计划判断”，这是用户最初问题的必要部分，已接受。没有需要用户先决定的技术分歧。

## 差异

| 编号 | 差异 | 处理 | 依据 |
|---|---|---|---|
| D1 | 主稿只定义当前重点，没有保存用户离开时确认的目标、完成情况和原下一步 | **接受问题，调整存储。** 新增项目级 `ProjectCheckpoint`，不挂到任意工作项事件 | `WorkEvent.workItemId` 必填，只能表达工作项事件：`shared/types/knowledge.ts:63-70`；恢复状态属于项目：`docs/product/001-resume-project-state.md:47-57` |
| D2 | 主稿没有正式定义“原计划继续、调整、信息不足” | **接受独立稿。** 加入 `PlanAssessment`，判断必须引用事件或卡片 | 当前事件已有 decision、next_step_change、block、result：`shared/types/knowledge.ts:52-61`；无依据判断必须降为信息不足 |
| D3 | 主稿右侧只写“说明、依据、影响、建议”；独立稿要求进入项目时先显示恢复说明 | **接受独立稿。** 项目为中心时，右侧固定先回答离开状态、后来变化和原计划判断 | 用户首先要回答“上次在做什么”；通用项目介绍不能完成这个任务 |
| D4 | 独立源码研究建议负责人使用结构化 Agent 身份；主稿只用 `event.actor=agent:*` | **第一阶段保留主稿。** 只交付一个受控 Agent，暂不增加容量、在线状态和 Agent 注册表 | 当前只有 `assignee: string` 和 `actor: string`：`shared/types/knowledge.ts:29-47,63-71`；没有第二个真实 Agent 使用者，先加完整模型会扩大范围 |
| D5 | 主稿时间线未明确区分当前执行和历史结果 | **接受独立稿。** 当前 doing/blocked/confirmed 放在“现在”，done/cancelled 和旧结果归入历史；关键事件不聚合隐藏 | 当前状态与事件已经可独立推导：`shared/types/knowledge.ts:17-23,52-71` |
| D6 | 主稿 Agent 成功后直接把工作项置为 confirmed；独立稿强调失败和验证也要可回看 | **合并。** 成功写 result 并置 confirmed；失败写 block；用户验收后再进入 done | `confirmed` 现有标签就是“待确认”：`shared/types/knowledge.ts:140-147` |
| D7 | 主稿新增 Project 与 projectId；独立稿一致，但强调旧数据迁移 | **一致。** 读取时补默认项目，写回时不要求一次性重写所有旧 JSON | 当前对象没有 projectId：`shared/types/knowledge.ts:5-47`；现有 JSON 文件必须保留 |
| D8 | 参考图展示多个 Agent 和项目统计，可能被当作必须实现 | **明确不实现假数据。** 只显示真实对象、真实事件和一个真实受控 Agent | 用户要求业务逻辑优先；规格已禁止为图片伪造对象和结果 |
| D9 | 小屏画布如何处理 | **采用顺序布局。** 1280×800 保持完整桌面交互；更窄时按画布、说明、时间线排列，不承诺完整图操作 | 参考图是桌面工作台；自由缩放不在本轮范围 |

## 写入最终规格的修改

1. 在 `ProjectCanvasSnapshot` 中增加恢复说明和原计划判断。
2. 定义用户确认状态记录的写法，以及没有确认记录时的提示。
3. 项目为中心时，右侧先显示恢复说明。
4. 时间线区分当前执行和历史结果。
5. 单元测试增加状态记录、三态判断和关键事件不被隐藏。

## 未解决但不阻塞实现

1. 何时自动建议用户确认一次离开状态，需要真实使用后决定；第一阶段由明确按钮触发。
2. 第二个真实 Agent 出现后，是否增加 Agent 身份、容量和在线状态模型；当前不提前设计。
3. 跨项目关系暂不展示，但数据模型不能阻止后续增加。
