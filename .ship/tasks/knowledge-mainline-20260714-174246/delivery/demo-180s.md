# 180 秒演示说明

## 0–20 秒：问题

一个人同时推进多个项目、材料和 Agent 工作。重新回来时，最耗时间的不是找一份文件，而是回答：上次做到哪里、后来变了什么、现在先做什么。

## 20–50 秒：进入项目

打开 `fc-opc-ibot`。中央是当前项目；Agent 只突出有事件或材料依据的当前重点。右侧说明当前目标、离开后的变化，以及原计划能继续、需要调整，还是信息不足。

## 50–80 秒：聚焦对象

点击一个当前重点。它成为中央对象，画布只展开一层直接相关的材料、工作项、关系和执行记录；右侧和底部时间线同时切换，URL 可保存和返回。

## 80–110 秒：核对来源

搜索“检索 来源”。结果直接显示会议、文档等来源；左侧显示这次检索命中、查看和关联过哪些材料。点击“检索验收标准”，它立即成为新中心。

## 110–145 秒：作出决定

创建或打开工作项，查看负责人、状态、下一步和直接依据。用户可以自己执行，也可以修改下一步后交给 Agent。系统拒绝没有负责人或下一步的进行中状态。

## 145–170 秒：执行写回

点击“交给 Agent”。Agent 读取当前工作项及直接依据，写回判断、缺口和下一步。右侧动态和底部时间线同步出现真实事件，刷新后仍然存在；外部 Agent 的结果会明确标为“外部 Agent”。

## 170–180 秒：结果

衡量的不是生成多少文字，而是恢复项目状态需要多久、错误启动了多少工作、多少决定能追到依据、多少执行结果能被重新找到。本演示已把回忆、查找、判断和继续执行放进同一页面。
