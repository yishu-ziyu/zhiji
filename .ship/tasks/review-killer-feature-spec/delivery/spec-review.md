## 0 · 综述(2 段)

**总判断:修正后过,不能直接进实现**。

**方向是对的**:选 C+A(单 Agent + 真决策动作 + Trace Panel)是 48h 黑客松的现实解法,优于 A 的"形式 > 内涵"和 B 的"复杂度失控"。但 spec 内部至少有 **3 个 ship-stopper**、**7 个评审扣分点**、**8 个 polish 项**,最高优先级是**与现有 DEMO_SCRIPT.md 合并成单一 3 分钟走查**——spec 画了一套独立 180s,跟 narrative-framework.md 既定的脚本在数学上不兼容,现场只能跑一份。

3 个 reviewer 视角互补:① 评审工程 vs 用户工程的撕裂;② 真实留存 vs 现场打分的张力;③ step-3.7-flash 兼容性 vs "形式升级"的隐性技术债。**最该修的不是 spec 的复杂度,是对"真 Agent"叙事的硬撑**。

## 1 · P1 ship-stopper(必须修才能进实现)

### P1-A · Demo 走查与现有 DEMO_SCRIPT.md 数学冲突
- **位置**:§3.4「3 分钟 Demo 走查」+ §五「7 天倒排」
- **冲突**:DEMO_SCRIPT.md 已锁定 180s(开场 30s + 电商 90s + 效率 45s + 收尾 15s),spec 又写了一套独立 180s(画面 1 60s + 画面 2 60s + 画面 3 60s)。**两套加起来 360s,现场主持人只给 180s**
- **现场二选一,spec 没决定哪个**
- **修正**:
  - 选**合并方案** —— 开场 30s 保留(DEMO_SCRIPT.md 第一幕);中间 90s 替换为"小掌柜早报 + 决策 + Trace Panel";效率 45s 砍到 15s,只保留看板;收尾 15s 保留
  - 画出 180s 内**逐秒时间切片**,标出哪些秒砍、复用、换新内容
  - 替代方案:把"小掌柜"开场和收尾挂到 DEMO_SCRIPT.md 第一幕 30s 和第四幕 15s,**中间 90s 不动**

### P1-B · step-3.7-flash tool_use 兼容性未验证
- **位置**:§三 3.5「与现有架构兼容性」+ §五 D4
- **漏洞**:spec 写"不依赖 tool_use 协议,只用 CoT 文本分析" —— 这不是工程决策,是**被兼容性逼出来的投降**
- **已知事实**(从 memory):step-3.7-flash 是 StepFun 的 reasoning 模型,拒收 `response_format/temperature/reasoning_effort`;adapter.ts 走 Anthropic 协议但**完全不解码 `tool_use` block**
- **最坏情况**(概率 30-40%):step-3.7-flash 不支持原生 tool_use,Trace Panel 只能用 prompt engineering 假装工具调用,资深评审 30 秒内能从 DevTools 看穿
- **修正**:
  - **D1 晚上或 D2 上午第一件事**:`curl` 一次带 `tools` 字段的请求给 step-3.7-flash,15 分钟出 Go/No-Go
  - Go → adapter.ts 加 `tool_use` block 解析(2-3 小时工作量)
  - **No-Go → 诚实降级**:Trace Panel 改叫"思考过程可视化",真决策改叫"AI 建议 + 一键执行",金句保留但措辞诚实
  - spec §五 D4 加一条:`validate step-3.7-flash tool_use by EOD 07-05`

### P1-C · D6 三件事挤一天,实施计划不可执行
- **位置**:§五 D6「第二个真决策动作 + 早报数据模型 + 9:00 推送逻辑」
- **漏洞**:三个新功能挤在一天,任何一件卡住 → D7 没 buffer
- **D7 三个不可压缩项**:UI polish + 录 demo 视频 + 演练 3 次
- **修正**:
  - D6 砍掉「第二个真决策动作」
  - 只做「早报数据模型 + 推送逻辑」
  - 把第一个真决策动作做深做透(支持批量把会议纪要待办拖进看板 + 自动分配负责人 + 截止日期)
  - **1 个深决策 > 2 个浅决策**

## 2 · P2 评审扣分点(不修会扣分)

### P2-1 · 金句体系与 narrative-framework.md "虚拟创始团队"撞车
- spec 提"AI 运营总监"作为金句,但 narrative-framework 已定调"虚拟创始团队"
- 评审会困惑"你说是 1 个 AI 总监还是 1 个 AI 团队",扣商业潜力 + 叙事张力
- **修正**:
  - 把"AI 运营总监"重新定义为"虚拟创始团队里的运营总监模块",把两句话捏成一句:"**一个 AI 团队,你的运营总监叫小掌柜,99 元/月**"
  - 或换用 narrative-framework §四 推荐的"Workshop"隐喻

### P2-2 · 失败模式清单里第 1 条"ChatGPT Wrapper 陷阱"的防御策略被破坏
- spec §一 1.2 写"4 个功能本质都是单轮 chat 包壳" + "prompt wrapper (用 GPT-3.5 + if-else 也能跑 80%)"
- **等于在评审面前亲口承认产品是 chat wrapper**,narrative-framework 的防御话术失效
- **修正**:
  - §一 1.2 整段改写为:"4 个功能当前是**独立 Workflow**(各自有 Intent Router + Prompt Template),杀手锏要做的是给它们加一个**统一的 Agent 编排层**,让 4 个 workflow 的状态可以**跨任务流转**"
  - 叙事从"我们之前是 wrapper,现在升级"变成"我们之前是 4 个 workflow,现在有了 1 个 Agent 把它们串成 Agent"

### P2-3 · 商业潜力维度没接 BUSINESS_MODEL.md 的财务数字
- spec §四只写"99 元/月锚点完整""金句最强",没引 ARPU、MAU、退出路径
- 商业化奖项评委(投资背景)会直接追问"99 块的获客成本多少?LTV 多少?为什么不是 39?"
- **修正**:§四商业潜力那一行扩成 3-5 行,引用 BUSINESS_MODEL.md 的关键数字(5000 MAU × 10% × 150 × 12 = 90 万)+ 加一行"**99 元/月 = 500 次 LLM 调用 + 无限次预生成内容读取**"挡投资人追问

### P2-4 · 「复用在 2026 黑客松是减分项」与「嫁接 4 个旧功能」自相矛盾
- spec §二 阶段 3 第 2 条:复用是减分项
- spec §三 3.2:方案核心是嫁接到 4 个旧功能
- 评审一句"你这不就是把已有功能串一下吗",spec 没逻辑盾牌
- **修正**:
  - §二 阶段 3 第 2 条改写为"**简单功能复用**是减分项,但**跨功能状态编排**(把 4 个独立 workflow 升级为 1 个 Agent 编排层)是加分项"

### P2-5 · 「自动加入看板」是 UI 自动化,不是商业决策
- spec 推荐的"自动加入看板"= 把会议纪要 JSON 的待办拖进 Kanban,**这跟 Notion AI 自动建任务无差异**
- 评审追问"这跟 Notion 有什么区别"——答不上
- **修正**:**Q1 改为「自动调整选品库权重 + 一键确认」**——选品分析出 SKU-12 是潜力款,小掌柜自动把它从"观察库"挪到"主推库" + 预生成 3 条脚本草稿 + 在 Kanban 自动建"今晚发布 SKU-12 视频"任务。**用户点"确认"一次完成 4 个动作**,有商业后果(主推库影响明日算法权重)

### P2-6 · Trace Panel 暴露的"审计能力"对 OPC 用户 over-spec、对评审 under-spec
- spec Q3 在「全文 CoT」vs「3-5 句关键点」二选一,丢失最优解
- **修正**:**默认折叠 3-5 句关键点 + 点击 Thought 卡片展开全文 CoT** + 加一个"为什么这样想"问号(悬停弹 1 句话)。这是 GitHub Copilot Workspace + Cursor 的现行做法

### P2-7 · "小掌柜"在评审眼里是"另一个聊天机器人"的风险
- 媒体/创客评委问"义乌张薇打开 iBot 第一眼看到右下角圆头像亮灯,会不会觉得'这是什么鬼'?"
- spec 完全没提"可关闭性"
- **修正**:加一条"**用 2 次后小掌柜可折叠成右下角极小图标,不打扰工作流**"。这是 Notion AI "用 3 次后变灰角"的现行做法

## 3 · P3 polish(改不改都行,但改了更好)

### P3-1 · "AI 运营总监"措辞涉嫌《广告法》风险
- "雇"字用在 AI 服务上有虚假宣传嫌疑
- **修正**:改"AI 运营助理"或"AI 运营总监模块"

### P3-2 · "AI 运营总监"金句被反讽检验
- 挑剔的评审会说:"这不是雇佣,这是工具;你不是雇主,你是客服"
- **修正**:备选金句"**你的 AI 不是回答问题,你的 AI 是开会——它在会上做决策,你在会上签字**"

### P3-3 · 小掌柜视觉:用 emoji 不用 CSS 剪影
- CSS 几何剪影 5 秒内不识别
- **修正**:用「戴帽子的店主 emoji」(👨‍💼 或 🧑‍💼)+ 状态灯。emoji 是评审熟悉的视觉语言,3 秒可识别

### P3-4 · 风险预案只覆盖"LLM 慢",缺另外 3 个常见事故
- **缺**:Trace Panel 动画太快评审走神;确认按钮误点;低端评审电脑/投影卡顿
- **修正**:风险预案扩到 3-4 条,加 `prefers-reduced-motion` 降级 + 早报滑入可关闭 + 90s 完整录像兜底

### P3-5 · 1.2 段自爆式定性"prompt wrapper (用 GPT-3.5 + if-else 也能跑 80%)"
- 见 P2-2 修正建议,整段改写

### P3-6 · 实施计划没做用户验证
- spec 7 天倒排里没有"找 2-3 个真实 OPC 用户看 demo + 问愿不愿付 99"
- narrative-framework §三"必须锚定的三个真实感来源"里要求"具体人物画像(义乌 27 岁张薇)"——spec 没接
- **修正**:D6 晚上跑 2 个非项目人员的 user test(远程看 30s demo + 问付费意愿)

### P3-7 · 实施计划没明确 D7 EOD 实际能 ship 的最小子集
- spec 描述的是最终态,不是实现路径
- **修正**:写一份 `docs/superpowers/specs/2026-07-04-minimum-viable-subset.md`,**列清楚 D7 实际能 ship 的功能 vs spec 画的最终态**,砍掉的明面写出来

### P3-8 · "自动调整选品库权重"是 narrative-framework 隐喻的最佳落点
- narrative-framework §四"空军地面引导员"强调"该打的时候打、该撤的时候撤"的决断感
- 自动调整选品库权重 = 把"主推 / 观察 / 弃用"三档切换 = 空军引导员选择打击目标
- **修正**:把 Q1 推荐和 narrative-framework 的"空军地面引导员"隐喻对齐

## 4 · 决策论证反思(C+A vs A vs B 的论证漏洞)

### 漏洞 1:对 B 的"12+ 人天复杂度"是估算错误
- B 方案的真实复杂度不在多 Agent 编排,而在"3 个 Agent 工位动画 + 状态灯同步"
- 砍掉动画 = 砍掉 60% 工作量,B 可压到 6-7 人天
- **spec 没做敏感性分析**

### 漏洞 2:对 C 的补丁方向反了
- spec 给 C 加"真决策动作",但本质是"让建议看起来像决策"(确认按钮还是用户按)
- 评审追问"跟 SaaS 表单有什么区别",补丁线塌陷
- **真正打 C 的补丁是"自动监控 + 自动触发"**(无需用户先点确认)

### 漏洞 3:金句借方案 C,论证自洽性弱
- spec 把金句外包给 C 的"AI 运营总监",等于"我做的杀手锏,金句来自另一个候选"
- **解决方案**:杀手锏自己出独立金句

### 漏洞 4(最致命):"真决策"在 spec 里被消解
- §三 3.3 表给的是 3 个浅候选(自动上架/自动发话术/自动加入看板),**3 个都没有商业后果**
- 为了规避过度承诺风险,选了最弱的决策形态
- **真正不被消解的"真决策"是"自动调价/自动调预算"** —— 但 spec 主动放弃了

## 7 · 最严厉的 3 个攻击(评审视角)

### 攻击 1(投资人视角):"99 元的获客成本?LTV?为什么不是 39?"
- 致命点:99 元是 SaaS 订阅还是 API 额度?超出怎么算?两难陷阱
- 挡法:spec 加"99 元/月 = 500 次 LLM 调用 + 无限次预生成内容读取",引 BUSINESS_MODEL.md 的毛利模型

### 攻击 2(技术派 PM 视角):"Agent Loop 是真 Loop 还是 prompt 包装?"
- 致命点:spec §七自己说"我们不依赖 tool_use 协议"——等于自爆
- 挡法:把"用 CoT 文本解析"作为 fallback(不是首选),D1 晚上先 spike 验证 step-3.7-flash 真支持 tool_use

### 攻击 3(媒体/创客评委视角):"这个杀手锏是给投资人看的,还是给义乌张薇看的?"
- 致命点:小掌柜头像 + 状态灯 + Trace Panel 对真实小商家是视觉噪声
- 挡法:明确"用 2 次后小掌柜可折叠成右下角极小图标"
