# Problem and Solution

## Problem Summary

一人公司在客户对话后产生大量可执行承诺，但没有编制去当项目经理。

注意力有限 → 承诺漏记 / 未拆任务 / 无人催 / 无确认 → 交付失控。

传统「会议纪要」只优化文字整理，不优化说到做到。

## Severity and Frequency

- **Severity**：漏一项硬承诺 = 返工、信任损伤、收入延迟（OPC 无冗余缓冲）
- **Frequency**：每次客户沟通、对齐、口头改需求都会发生
- **Who feels it**：独立开发者 / 顾问 / 超级个体（自用即主用户）

## Solution Idea

交付闭环 Agent（运营助手）：

```text
对话/需求文本 → 承诺捕获 → 任务与截止 → 推进与逾期暴露 → 交付说明 → 客户确认
                              ↑________________ 仪表盘闭环率 ________________↑
```

人只做关键拍板；系统负责记忆、拆解、盯梢、可评测数字。

## Evidence

- 用户自述工作流：问题 → 分析 → 开发 → 交付；不要纪要
- 赛事手册赛道二关键词：工作、运营、**交付**
- Solo founder 外部叙事强调 ship / feedback loop，而非内部会议文档
- 评分 90% 人工：场景真 + 体验 + 可演示闭环

## Non-goals

- 纪要美观度竞赛
- 电商经营数据闭环（无账号）
- 组织级权限与多人协作主路径
