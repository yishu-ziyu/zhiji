## 我们在做什么

页面：`/track/knowledge`（唯一入口；旧 `/mvp` 永久重定向）

**北极星形态（Canvasight 参考图）：**

| 区 | 内容 |
|----|------|
| 左 | 多项目导航 |
| 中 | 以当前项目为中心的关系画布（真实节点，可点） |
| 右 | 当前对象的摘要 / 依据 / Agent 与项目动态 |
| 底 | 与焦点相关的时间线 |

授权本地夹、拖入材料是**喂料**，不是产品终点。终点是：进入项目后看见真实结构，点开有依据，能决定下一步。

一句话：

> 重新进入项目时，不必靠记忆找进度：系统指出当前重点及依据，用户沿直接关系理解情况、作出决定，并继续追踪人或助手写回的结果。

**不做：** 当编辑器卖；接微信私聊；第二套产品面或 MVP 分叉；先假热闹再真结构。

## 词怎么用

| 说法 | 意思 |
|------|------|
| 知识工作者 | 靠检索、笔记、会议材料干活的人；先服务你自己 |
| 知识卡片 | 一条能以后再用、能说来源的记录 |
| 来源 | 会议 / 邮件 / 聊天 / 文档 / 手记 |
| 关系 / 边 | 两张依据卡之间可解释的连接；必带来源句 |
| 来源句 | 支持这条边的一句可指回文字 |
| 工作项 | 一件事：状态 + 负责人 + 下一步 + 时间线 |
| 事件 / 时间线 | 评论、决定、状态变化、阻塞、结果写回 |
| 状态 | 待开始 → 进行中 → 待确认 → 完成；可阻塞 |
| 本项目材料 | 项目下可打开的本地文件（先 Markdown 阅读） |
| 失败 | 结果没来源、重启丢数据、状态点不动、中心被测试/噪声标题占满、和记笔记软件说不清差别 |

## 已定的事

1. 只做效率赛道里「检索 / 知识 / 协作」相关能力。
2. 产品就是上面那一套；客户改约定不再当产品；不主打微信私聊。
3. 模型可以帮理解、拆任务；**写进库的卡片和状态算数**；人点状态。
4. 打开工作台（URL 无项目）→ **最近在弄的项目**；界面**优先一条**「请先看」；其它区域更安静。
5. **中心是过滤器**：默认不把测试标题（e2e、冒烟、smoke）或内部噪声放在注意力中心。
6. **本项目材料**可列表、可打开阅读（Markdown 先做）；不把全库文件铺满画布中心；第一版不以向量库当知识本体。
7. **Agent 双记忆：** **项目记忆**（`shared/project-memory`：事件 + 不可变理解 + CAS）负责长期真源；**对话记忆**（`shared/agent-memory`：会话回合 + 未完成工具意图）负责和用户聊天接话。二者互补，对话不能改写已接受理解。语义/向量检索只可作附属手段，不作记忆本体（见 `docs/decisions/2026-07-17-agent-dual-memory.md`）。
8. 换项目：左侧列表可用；也可按项目名命令跳转（如 ⌘/Ctrl+P）。
9. 依据卡之间的关系：类型 + 来源句；画布只展一层直接关系。
10. 改动必须降低理解成本；加噪声的不发。
11. 有单测；浏览器里要能走通关键路径。
12. 代码只在 `main`：`/Users/mahaoxuan/Desktop/黑客松/fc-opc-ibot`。目录见 `docs/product/代码在哪.md`。
13. 视觉：系统蓝 `#007AFF`，Plus Jakarta Sans；SaaS 组件可用。

## 第一纵深验收（阶段 2）

当你授权夹或拖入真实项目后：

1. 看见四区工作台（左项目 / 中画布 / 右说明 / 底时间）
2. 中心是当前项目
3. 周围至少 3 个来自材料、工作或理解依据的**真实**可点节点（不是占位假数据）
4. 点任一节点：右侧出现标题、摘要，材料类能点到依据

工程入口：

- 画布图：`shared/knowledge/project-canvas.ts` → `buildProjectCanvasSnapshot`
- 页面：`app/track/knowledge/page.tsx` + `components/ProjectCanvas.tsx`
- 授权夹：`shared/project-memory/` → connect 后须把可读材料落到 knowledge 材料/卡片，画布才能亮
