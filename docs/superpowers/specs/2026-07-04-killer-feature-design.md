## 一、问题陈述

### 1.1 已对齐的事实(来自项目材料)

| 项 | 数据 |
|---|---|
| 黑客松时间 | 2026-07-11 ~ 07-12 杭州 · 48h |
| 今日(本 spec 撰写时) | 2026-07-04 · 剩 7 天准备 |
| 项目已有 | Next.js 16 + shadcn/ui + step-3.7-flash + 5 API + 3 prompt + 11/11 E2E |
| 现有 4 个 Must | 电商(选品分析/短视频脚本/私域话术) + 效率(会议纪要/项目看板) |
| 叙事核心 | 一人公司 = 1 个 AI 团队替代 24 万/年,99 元/月订阅 |
| 评分维度 | 完成度 25% / 应用价值 25% / 商业潜力 20% / 创新性 20% / 安全性 10% |

### 1.2 现有 4 个功能的"隐藏弱点"

经质询确认,4 个功能本质都是**单轮 chat 包壳**:
- 每次都是用户主动问 → LLM 回复
- 没有跨任务的状态流转
- 没有"AI 替我做了什么决策"的体现
- 在资深评审眼里 = prompt wrapper(用 GPT-3.5 + if-else 也能跑 80%)

### 1.3 用户原始诉求

> "加一个杀手锏功能,需要充分调研和讨论。"

**核心需求解读**:在 7 天窗口内,通过 1 个新能力让 demo 在评审眼里**从 chat 升维到 Agent**,同时不破坏已有投入。

## 二、调研方法

| 阶段 | 投入 | 输出 |
|---|---|---|
| 阶段 1 | 派 3 个 Agent 并行调研不同立场 | 3 份候选杀手锏方案 |
| 阶段 2 | 派 3 个 Agent 互相质询 | 暴露每个方案的真实弱点 |
| 阶段 3 | 综合诊断 → 组合方案 | 选定最终方向 |

### 阶段 1:三个候选方案(各 Agent 立场)

| | 实战派黑客松老兵 | OPC 叙事派 PM | Agent 形态派架构师 |
|---|---|---|---|
| 方案 | Agent Loop Live | AI 团队作战室 | 小掌柜日报 |
| 核心画面 | Thought→Tool→Obs 卡片流 | 大楼 + 3 Agent 工位 | 早 9 点推送 + 自动决策 |
| 主攻评分 | 创新性 20% | 创新性 + 商业潜力 | 商业潜力 + 应用价值 |

### 阶段 2:三个 Agent 互相质询(关键攻击)

**对方案 A(Agent Loop Live)的攻击**:
1. 评审看不出"过程透明"的价值,只 tech 评审 buy in
2. 金句空白,无一句话能复述
3. LLM tool_call 调试是真正无底洞,2.5 天估算是低估

**对方案 B(AI 团队作战室)的攻击**:
1. 复杂度失控(估 6 人天,实际 12+),3-Agent 状态机风险叠加
2. 是 Flowith Matrix(2026-07-02)的简化版,易被认成复刻
3. **砍掉项目看板是砍错了**(三个 Agent 都同意)

**对方案 C(小掌柜日报)的攻击**:
1. 没有"哇时刻",日报 + 自动回复 + 跳转听 30 秒完
2. **"AI 运营总监"是过度承诺**(三个 Agent 都同意),没真决策权
3. 复用已有功能在 2026 黑客松是减分项

### 阶段 3:综合诊断

**三个 Agent 的共识信号**:
1. 真的 Agent 必须能做决策(不只给建议)
2. 复用在 2026 黑客松是减分项
3. 48h 内不能花 > 6 人天

**为什么选组合方案 C + A**:
- C 的商业叙事最完整 + 风险最低(完成度 25% 稳)
- 但 C 的核心弱点是"无真决策" → 补 A 的"可追溯 trace"+ **加 1 个真决策动作**
- 不选 B:虽然画面震撼,但复杂度失控 + 砍看板不对 + 形式>内涵

## 三、最终方案设计

### 3.1 方案名

**「小掌柜」AI 运营总监 + Agent 实时追溯看板**

副标题:侧边栏常驻 + 跨轨道触发 + 真决策动作 + 完整可审计 trace

### 3.2 核心定义

> 不是新功能。是把现有 4 个功能 + 1 个新能力**(自动跨功能决策)+ 1 个新体验**(实时追溯面板)组合成一个"看起来是 Agent 而不是 chat"的统一形态。

### 3.3 三个核心组件

#### 组件 1:侧边栏常驻「小掌柜」AI 头像

**形态**:任何页面右下角都有一个圆头像 + 状态灯(绿:空闲 / 黄:工作中 / 红:需决策)
**交互**:点击展开 = 弹出对话面板(可对话、可看历史);点头像 = 触发新决策任务

**9:00 自动推送**(每日早报,**不是 push notification,是用户进入页面时第一眼看到**):
> "老板,昨晚 23:40 用户 @林小姐 私信问 XX 蓝牙耳机续航,已用话术库 #EP-07 自动回复;今早淘宝热搜新词『骨传导耳机 2026 款』匹配你的选品库 SKU-12,建议今日主推;昨天上线 3 个 SKU 中,SKU-09 加购率 2.1% 远高于平均 0.8%,建议今晚追加投放 50 元。"

#### 组件 2:Agent Trace Panel(实时追溯)

**触发**:小掌柜执行任何决策时,自动展开
**形态**:右侧浮层,纵向时间轴,每步一张卡片

**卡片类型**:
- 💭 **Thought**:小掌柜在思考什么(LLM CoT 透明化)
- 🔧 **Tool Call**:调用了什么工具(JSON 参数 + 工具名)
- 📊 **Observation**:工具返回了什么(数据/状态)
- ✅ **Decision**:做出了什么决策 + 为什么(可审计)

**核心金句**:
> "Agent 不是黑盒,我每一步怎么想、怎么干、为什么这个结论,你都看得到。"

#### 组件 3:真决策动作(关键 — 让它从 Chat 升维到 Agent)

**至少一个真决策,不是"建议"**:

| 决策类型 | 触发条件 | 动作 | 兜底 |
|---|---|---|---|
| 自动上架 SKU | 选品分析 ≥ 80 分 → 小掌柜说"建议上架" → 用户点"确认" | 写入 mock 数据库 + 显示"已上架" | 用户可一键撤回 |
| 自动发私域话术 | 客服触发"XX 商品咨询" → 小掌柜选话术模板 → 用户点"批准发送" | 显示"已发送" + 实时反馈标签 | 5 秒内可撤回 |
| 自动加入看板 | 会议纪要生成后 → 小掌柜自动建卡片到 Kanban | 卡片出现在"待处理"列 | 用户可拖动修改 |

**关键设计哲学**:**Agent 不是替老板决定,是把"建议"变成"可一键批准的执行"**。这不是过度承诺,因为有"确认按钮"和"撤回机制"。

### 3.4 3 分钟 Demo 走查

**画面 1(60 秒·早晨)**:
- 进入 iBot,右下方小掌柜头像亮起"工作"灯
- 自动弹出早报消息(滑入动画)
- 评审看到:这不是 chat,这是有"组织"的 AI 员工

**画面 2(60 秒·决策)**:
- 用户点早报里"建议主推 SKU-12"
- 小掌柜自动展开 Trace Panel
- 卡片依次出现:Thought(分析热度数据)→ Tool(`fetch_taobao_hot`)→ Observation(数据)→ Thought(匹配 SKU)→ Decision(主推建议)
- 用户点"采纳建议"
- 自动跳到选品分析页 → 已有 SKU-12 的预填选品报告
- 评审看到:这有过程、有工具、有状态流转 = Agent

**画面 3(60 秒·落地)**:
- 在选品报告页点"生成脚本" → 短视频脚本自动生成
- 再点"加入今日待办" → 看板出现"发布 SKU-12 短视频"卡片,带截止时间 23:59
- 评审看到:跨轨道、跨工作流、有 deliverable

### 3.5 与现有架构的兼容性

**完全兼容(不需要重写)**:
- ✅ step-3.7-flash + 流式输出(架构已定)
- ✅ Intent Router(加一个新的 IntentType:"agent_decision")
- ✅ Workflow Registry(新加一个 agent workflow 而已)
- ✅ shadcn/ui 暗色主题

**新依赖**:**0 个** — 全用现有 shadcn 组件 + Tailwind

| 评分维度 | 占比 | 这个方案的得分逻辑 |

## 七、为什么不是其他 2 个方案

### 为什么不选 Agent Loop Live(方案 A)

| 攻击点 | 我们的回应 |
|---|---|
| 评审看不出过程透明的价值 | 我们保留 process visible,但把它作为**审计能力**,不是核心卖点 |
| 金句空白 | 我们用 C 的"AI 运营总监"金句 |
| LLM 工具调用调试是真正无底洞 | 我们**不依赖 tool_use 协议**,只用 CoT 文本分析(因为现有架构是 step-3.7-flash,不是 Anthropic) |

### 为什么不选 AI 团队作战室(方案 B)

| 攻击点 | 我们的回应 |
|---|---|
| 复杂度失控(12+ 人天) | 我们只做单 Agent + 状态灯,**不画大楼** |
| 是 Flowith 简化版 | 我们没有 3 个 Agent 工位动画,不会被认成复刻 |
| 砍看板是砍错了 | 我们**保留看板**,把它升级为"决策落地"载体 |
