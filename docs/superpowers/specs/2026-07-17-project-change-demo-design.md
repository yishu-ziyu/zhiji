# 项目变化情报 Demo 设计

## 目标

在知几已授权的真实项目文件夹中放入一份新文件后，桌面 App 自动观察到变化，调用已连接的真实模型判断它是否影响当前项目，并把“发现变化 → Agent 判断 → Owner 决定”显示在底部时间线。

## 用户路径

1. Owner 打开已授权项目，知几恢复该文件夹的只读观察。
2. Owner 在 Finder 中把一份明显不相关的文本或 Markdown 文件放进项目文件夹。
3. 知几显示“检测到项目变化，正在判断”，只使用授权边界内的真实文件运行 Agent。
4. 时间线先出现真实文件变化，再出现 Agent 的候选判断；若模型判断无关，文案明确建议忽略或移出，但仍是待 Owner 确认的候选。
5. Owner 对 Claim 选择接受、修改、拒绝或暂缓，真实裁决进入时间线。

## 数据与信任边界

- 文件变化来自 `@parcel/watcher` 写入的 `ChangeEvent`，不制造演示记录。
- Agent 结果来自当前 Run 对应的 `UnderstandingRevision`，失败时只显示失败，不生成判断。
- Owner 决定来自持久化的 `OwnerResolution`。
- 时间线只是上述真源的只读投影，不另建第二套事实库。
- 首次恢复项目时把已有事件作为基线，不因历史文件变化自动重跑；只处理本次打开后新增的非 `reconciled` 事件。

## 最小实现

- `agent-session` 恢复接口在冷启动时重新挂载已持久化授权夹的 watcher，并返回 Run 的真实时间和事件范围。
- 工作台每 2.5 秒读取轻量 session；出现基线之外的真实增量事件时只触发一次 `source_change` Run。
- 自动 Run 的 Owner 上下文明确要求比较新文件与当前项目目标；无关时直说“无关”并给一个处置选择。
- 时间线合并三类真实里程碑：文件变化、Agent 候选判断、Owner 裁决。气泡直接显示业务文案，不显示内部 ID。

## 不做

- 不扫描授权边界外的文件。
- 不把普通聊天自动写成项目事实。
- 不自动接受 Agent 判断或创建正式任务。
- 不为 Demo 写硬编码成功结果。
- 不展开完整后台守护进程、系统通知或跨崩溃 checkpoint。

## 验收

打开已授权项目 → 往授权夹新增 `与本项目无关的旅行清单.md` → 页面自动出现真实变化 → Agent 给出带文件依据的“与项目无关/不影响当前目标”候选 → 底部时间线能直接看到该判断 → Owner 暂缓后时间线出现真实决定。
