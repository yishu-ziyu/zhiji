# Canvas Mini Map 悬停展开设计

## 目标

在不移除画布方位感的前提下，让 Mini Map 默认退居为左下角的极小缩略图，只在用户表达导航意图时展开。

## 已确认方案

- 默认保留约 `56 × 42px` 的可见缩略图。
- 鼠标移入、键盘聚焦或触屏点击后，从左下角展开到 `200 × 150px`。
- 展开后保留 React Flow 原有的拖动平移和滚轮缩放。
- 移出后短暂延迟再收起，避免指针在边缘时闪烁。

## 动效

- 只过渡 `transform` 和 `opacity`，不动画 `width` / `height`，避免画布重排。
- 使用左下角 `transform-origin`，形成空间上连续的展开。
- 展开与收起控制在 300ms 以内，且可在中途反向。
- `prefers-reduced-motion: reduce` 时取消位移过渡，保留直接状态反馈。

## 结构

- 将现有 `MiniMap` 放入一个专用的导航外壳。
- 外壳负责收起、展开、聚焦和无障碍语义；`MiniMap` 继续只负责映射画布。
- 不改节点、边、布局或项目数据。

## 验收

1. 打开项目画布 → Mini Map 默认只占左下角小块。
2. 鼠标移入 → Mini Map 平滑展开，不推动画布。
3. 鼠标移出 → Mini Map 稍后收起，不闪烁。
4. 键盘聚焦或触屏点击 → 同样可展开。
5. 展开后拖动或滚轮 → 画布视口正常响应。
6. 开启“减少动态效果” → 无缩放动画，功能仍可用。

## 测试

- Playwright 先断言 Mini Map 默认宽高显著小于完整尺寸，当前实现应先失败。
- 实现后断言悬停展开到完整尺寸，移出后恢复。
- 运行相关 E2E、TypeScript/ESLint 检查，并在真实预览中进行视觉复核。

## 范围外

- 不改画布导航模型。
- 不新增持久化偏好。
- 不为 Mini Map 增加新按钮或独立设置页。
