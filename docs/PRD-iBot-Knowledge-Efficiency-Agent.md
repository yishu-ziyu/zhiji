# iBot Knowledge Loop Agent - PRD

**版本：** 0.2 freeze  
**日期：** 2026-07-14  
**主线：** 知识闭环（用户冻结）  
**完整制品：** `.ship/tasks/knowledge-mainline-20260714-174246/product/08-prd.md`

## 1. 产品定位

面向**知识工作者**的 Agent 辅助闭环：

**资料检索（带来源）→ 知识卡片（可复用）→ 行动项（可推进状态）**

不是 Notion 编辑器，不是微信交付 CRM。

## 2. 目标用户

- 首要：本人（知识工作者）
- 次要：同赛道一人公司 / 小团队知识岗（叙事）

## 3. 核心痛点

1. 材料散在聊天、纪要、文档里，需要时找不回  
2. 找回来的内容往往**没有来源**，不敢当事实  
3. 有结论但落不成**可勾选、可验收**的下一步  
4. 工具很多，但检索 / 沉淀 / 行动不在同一对象上

## 4. 核心能力

| 能力 | 说明 |
|------|------|
| 检索 | 关键词 + 来源过滤；结果展示 source |
| 沉淀 | 手记 / 会议粘贴 / 目标拆解 |
| 行动 | todo→doing→confirmed→done |
| 工具面 | `/api/knowledge/mcp` 五工具 |
| 持久化 | `data/knowledge/*.json` |

## 5. 验收（P0）

见完整 PRD A1–A3：

- 能搜到带来源的卡  
- 新卡可持久化  
- 行动状态可推进并保持  

## 6. 成功指标 / 杀死条件

见 `.ship/tasks/knowledge-mainline-20260714-174246/product/08-prd.md`。

## 7. 明确不做

- 客户变更主 demo  
- 个人微信读私聊  
- 完整向量库卖点  
- 多租户权限  

## 8. 金脚本

打开 `/track/knowledge` → 搜「检索 来源」→ 推进一条行动 → （可选）入库 → 一句话：「搜得到、收成卡、能推进。」
