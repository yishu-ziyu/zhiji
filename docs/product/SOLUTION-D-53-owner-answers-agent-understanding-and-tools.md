## 2. 产品决定（可执行）

### D-53a · 任务完成 vs 智能体生命

- **智能体**可持续监视、修正、生长（对齐 D-39）。  
- **一次具体任务**（例：授权夹 → 初始有来源理解）有 **可验收结果**：  
  - 当我授权文件夹并让 Agent 理解项目时，我应看到 **有来源的当前理解候选**（或诚实「哪些还不清楚」），而不是空聊或只扫文件列表。  
  - 仅换模型接口、仍单次瞎编 = **任务未完成**。

### D-53b · 中等模型先跑通

- MVP 主模型固定 StepFun **`step-3.7-flash`**（pin 见 D-51）。  
- 产品验收逻辑：**中等模型 + 本产品工具/记忆/回执** 必须能在授权夹内完成「理解」任务。  
- 换更强模型 = 提质，不是本波前提。  
- 模型仍可替换；**项目真相不属模型**（D-51）。

### D-53c · 不设「读文件条数硬顶」；改靠架构

- **否决**产品规则「读满 N 个文件必须停」。  
- **改为要求**（须调研并工程落地，见研究笔记）：  
  1. **先地图后深读**（结构/索引工具便宜，全文贵）  
  2. **先搜后读**（grep/符号/路径，再 exact revision 摘录）  
  3. **按需多步 tool**，直到 **够用来源支撑理解** 或 **诚实 unknown**（D-51/D-52）  
  4. **任务外侧验收**：是否形成有 pin 的候选理解（不是「读了多少文件」）  
  5. Owner **可见可中断**（D-52）  
  6. 进程级超时/崩溃恢复 = **基础设施安全网**，不得冒充「已经理解够了」  

### D-53d · 第一版只在授权夹

- 首版 tool 与理解任务 **仅授权项目根内**。  
- 公开网页 / 外部账号读 **后置**（仍服从 D-10/D-18 何时打开）。

### D-53e · 仍强制（未改）

- 候选理解须 **Owner 确认** 才变当前真相（D-15/D-43）。  
- 写/发/删/commit/扩盘/敏感源 **须确认**（D-52）。  
- 回执：model + tool + 停止原因（D-51/D-52）。

## 3. 第四点 · D-53e FINAL（Owner：第四点按建议）

**产品规则（一句）：**

> **第一次理解整个授权项目时，可以在夹内自由摸清全貌；之后盯某一件事时，默认先围着这件事和相关材料转，只有影响说不清时再在夹内扩大搜索，并说明为什么扩大。**

| 阶段 | Agent 该怎样 | 你看到什么 |
|------|----------------|------------|
| **进门 / 首次授权**（D-50） | 在授权夹内建地图、按需深读、Git 只读，直到能给出 **整项目级** 有来源候选理解（或 unknown） | 「这个项目现在大概怎样 + 依据」 |
| **回项目 / 有变化**（D-44） | 优先看 **变化了什么** 与已授权监视相关材料；不够再扩大 | 六问式：现在/以前/变了啥/为何/依赖/证据 |
| **推进某一件事**（D-36/D-42） | 工具默认围着 **这件事**；全夹乱扫不是默认 | 这件事的状态与证据，不是无关文件墙 |

**对齐：** D-42 · D-38 · 顶尖 agent「先 map/search 再精确读」。  
**不是**禁止再读全夹——默认聪明；扩大须理由进回执。  
**Owner 确认语：**「第四点按建议」（2026-07-16）。

## 5. 可见验收（当我做 X 应看到 Y）

1. 我授权文件夹并要求理解 → 得到 **有来源候选理解**（或清楚的 unknown），不是空聊天。
