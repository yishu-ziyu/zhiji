### 1. 真读项目夹（产品创新 · 严格把控）

- **问题：** 别人也能做「聊几句 + 假进度」。我们的差异是：在你授权的夹里，**真的**建地图 → 搜 → 读文件 → 再推理，结论要么有出处，要么老实说不知道。
- **你要看到的结果：**
  1. 授权夹之后，它先摸清目录，再搜关键词，再打开具体文件，最后才说话
  2. 右侧过程一步一步在动，不是装饰
  3. 给出的理解能点到具体文件/片段；点不开出处 = 不合格
  4. 读不动或模型挂了 → 报失败，**不许**悄悄换成假理解
  5. 你确认 / 改 / 拒绝之后，下次打开还记得
- **严格不过线（任一即 FAIL）：**
  - 没读任何真实文件就下结论
  - 有结论但没有可核对的出处
  - 页面上看不到它读了什么（只见最终一段话）
  - 模型挂了仍显示「读懂了」
  - 确认后刷新/重进就丢
- **工程（2026-07-17 整包已落地 · 等你验收）：**
  - 工具环 + 真模型默认；禁静默降级（失败 `status=failed`，不写假候选）
  - 强制可用出处：`agent-evidence`；无出处拒绝「读懂」
  - HITL 确认写回；切换项目可恢复
  - 门禁：**A1–A6 + L1–L5 全绿**（含 live L1/L2/L3）
