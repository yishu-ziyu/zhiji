> 状态：已定（Owner 2026-07-16）

> 不在范围：接上项目后的工作台主 UI（另案）；`shared/project-memory/agent-model-loop.ts` / reconstruct 系列（工程师2 锁定）

## 1. 问题

用户打开产品，若第一眼只有「选文件夹 / 确认权限」，不知道进了什么产品、选完会得到什么，授权成本高、易流失。

## 2. 目标结果

打开约 3 秒内应同时看懂：

1. **这是什么** - 产品名 + 一句结果  
2. **选完长什么样** - 结果预览（示意即可）  
3. **下一步做什么** - 一个主按钮：选择项目文件夹  

确认权限卡是**第二步**（选夹之后），不是冷启动第一眼。

## 3. 信息架构（两步）

### 步骤 A · 冷启动首页（`entry`，无 pending 选择）

```
[顶栏] 产品标识 + 名称「项目理解」

[主区 · 居中，可单卡或「上预览 / 下行动」]
  标题（结果句，不是任务句）
  副文（≤ 20 字词量级，说明授权边界一句话）
  结果预览区（静态示意，非真数据也可）
  主 CTA：选择项目文件夹
  次 CTA（有最近连接时）：继续上次
  信任一行：只读所选目录 · 不扫全机 · 理解需你确认
```

### 步骤 B · 已选文件夹确认（`review`）

保持现有确认卡气质（居中白卡）：

```
已选择项目文件夹
{文件夹名}
连接后只在该夹内读目录/材料/版本信息…
三条权限说明
[重新选择] [取消] [用这个文件夹]
```

### 步骤 C · 连接中（`pipeline`）

同一壳内显示真实进度面板（已有 AgentProcessPanel），不跳回空白权限页。

## 4. 文案契约（人话）

| 位置 | 要求 | 推荐默认（可微调，勿工程黑话） |
|------|------|--------------------------------|
| 产品名 | 短 | 项目理解 |
| 主标题 | 结果导向 | 回到项目现状 |
| 副文 | ≤20 词量级 | 在你授权的文件夹里，重建有依据的当前理解。 |
| 主按钮 | 动词 + 对象 | 选择项目文件夹 |
| 次按钮 | 有最近才显示 | 继续上次 |
| 信任行 | 三条内 | 只读你选的目录 · 不扫全机 · 理解需你确认 |
| 预览区标题 | 可选 | 连接后你会看到 |
| 预览示例行 | 示意 | 「现在怎样」摘要占位 + 2 条「有出处的判断」灰态 |

禁止：Task/HTTP/adapter/fixture/UUID/rootPath 等工程字出现在首屏。  
fixture 提示仅 `?fixture=1` 时可见且极短。

## 5. 结果预览（示意）规格

- **目的：** 让用户看见「选完夹之后屏幕会长什么样」  
- **形态：** 静态示意块即可（不必接真 API）  
- **至少包含：**  
  - 一块「现在怎样」标题 + 1～2 行灰态摘要  
  - 2 条「有依据」示意（可带假文件名样式，标明示例）  
- **禁止：** div 假终端整屏、假仪表盘堆砌、编号 01/02/03 三卡说教  
- **标注：** 预览区可用极淡「示例」字样，避免用户以为已连接

## 8. 验收（BDD · Owner / 主座验收用）

当我打开 `http://127.0.0.1:3331/track/knowledge`（未连接）：

1. **看见产品是什么** - 产品名 + 结果句，不是只有「选夹」  
2. **看见选完会怎样** - 结果预览区存在且可读  
3. **只有一个主动作** - 「选择项目文件夹」为主按钮  
4. **不扫全机等信任信息** - 一行内可见  
5. **点主按钮** - 系统文件夹选择（或现有 picker 流程）  
6. **选完后** - 进入确认卡（步骤 B），不是直接黑盒转圈无说明  
7. **确认「用这个文件夹」** - 进入真实连接进度  
8. **无工程黑话** - 首屏无 adapter/Task/HTTP 等  
9. **字体** - 正文清晰；无全站艺术字导致难读  
10. **HTTP 200** - 页面可打开；相关单测/e2e 若有则通过  

非目标本轮：

- 不改模型环 / reconstruct（工程师2）  
- 不重做接上后的三栏工作台信息架构  
- 不强制真实「现在怎样」数据（预览可为静态示意）
