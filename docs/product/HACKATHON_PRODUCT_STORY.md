# 知几 · 黑客松产品故事与三帧业务逻辑

> 用途：现场 5 分钟叙事 + 评委 Q&A 的单一真源。  
> 原则：只写当前可演示的能力；愿景单独标注，禁止混成「已完成」。

---

## 1. 一句话

**知几帮「同时扛多个项目、要对结果负责的人」，在重新进入项目时，用最少注意力完成：现在怎样 → 凭什么 → 我决定什么。**

不是又一个聊天框，也不是又一个网盘搜索。它是 **本地授权范围内的项目情报 Agent**：读真材料、给候选判断、把依据钉在画布上，人只拍板。

---

## 2. 帮谁 · 解决什么 · 怎么提效

| 维度 | 内容 |
|------|------|
| **人** | 小团队 / 一人公司的项目负责人、产品负责人、客户交付负责人 |
| **痛** | 离开几天后回来：材料变了、聊天散了、不知道信哪版、不知道下一步 |
| **旧办法** | 重翻文档 + 问同事 + 在聊天里复述 → 贵、慢、易漏关键冲突 |
| **新办法** | 授权本地文件夹 → Agent 观察/检索/精读 → 结构化简报 + 画布依据 → 人确认/改/拒 |
| **效率** | **少看**（噪声过滤）· **少错**（判断回链到真实文件版本）· **少拖**（一个可拍板的 nextDecision） |

可量化的现场说法：

> 从「重读半小时」压成「看一条有依据的判断 + 点一下确认」。

## 3. 三张图代表的业务逻辑（产品语义）

这三帧是 **同一条判断链路** 的三个节拍，不是三个无关功能。

### ① 发散成候选

**人问：**「我们为什么改了方向？」  
**Agent 做：** 读授权夹里的对话与材料 → 生成 **若干候选解释**（虚线「候选」节点）→ 每条候选挂着对话条数 / 材料数。  
**右栏：** Agent 理解摘要、对话片段、相关材料、下一步建议。  
**语义：** 先 **发散**，不急着定一个真，避免模型一口咬死。

### ② 点击后换中心

**人点** 某一候选（如「客户需求变化」）。  
**画布：** 该候选成为中心；周围展开 **参考 / 依据 / 冲突 / 影响** 等有类型的边。  
**右栏：** 为什么看它、可点证据原文、来源版本、生成行动。  
**语义：** 从发散进入 **聚焦审阅**；图不是装饰，是证据拓扑。

### ③ 结果回流并留下足迹

**人做完验证/行动后：** 画布留下 **路径足迹**（来源 → 判断 → 行动 → 结果）。  
**右栏：** 路径足迹列表 + **待你确认的知识更新**（原判断 vs 候选更新 → 确认 / 编辑 / 拒绝）。  
**语义：** Agent 的推断 **回流** 成人确认后的项目知识；不自动写成真相。

## 4. 当前产品 vs 三帧（诚实对照）

| 节拍 | 图里要有 | 现在能不能 | 比赛可演示的替代路径 |
|------|----------|------------|----------------------|
| ① 发散 | 画布上虚线「候选」节点 + 计数 | **部分** | 自然语言问 Agent → 右栏 **候选判断 / Claims HITL**；中央 pin **已读材料节点** 为「业务逻辑/候选依据」；快捷「业务逻辑」强制读→串→高亮 |
| ② 换中心 | 候选为中心 + 类型边网 | **部分** | **点材料/工作节点** 换焦点；card 焦点可看真实 relation；右栏标题/依据随焦点变 |
| ③ 足迹回流 | 路径足迹 UI + 知识更新卡片 | **部分** | **简报 Claims 确认/编辑/拒绝** 持久化；时间线/事件留下 Agent 与人的裁决；画布 notice 记录本轮形态切换 |

### 明确还没到图里的部分（补齐 backlog，勿当已 ship）

1. 画布一级节点类型 = **假设候选**（虚线紫框、对话/材料计数 badge）  
2. 候选点击后 **按关系类型展开** 的证据网（参考/依据/冲突/影响 的固定版式）  
3. 右栏 **路径足迹（本次判断）** 时间轴组件 + 与画布边高亮联动  
4. 「生成行动」一键把候选变成工作项（若有需 HITL）

比赛演讲话术：

> 我们证明的是同一条业务闭环的 **骨架**：读真材料 → 候选判断 → 人确认 → 状态保留。  
> 三帧是北极星交互；当前 Demo 用「右栏候选 + 画布材料链 + 焦点切换 + Claim 确认」跑通同一故事。

## 5. 现场 Demo 金路径（对齐三帧故事）

1. **授权** 项目文件夹（本机，Key 已配好）  
2. 顶栏模型：可切 **本机代理 · Grok 4.5** / GPT / MiniMax M3 / Step 3.7 Flash（一键切换，无需再填 Key）  
3. 右栏说：**「展示业务逻辑」** 或 **「我们为什么改了方向？」**  
4. 看见：工具读文件 → 简报/候选 → 中央节点高亮/串联  
5. **点** 某个材料节点 → 右栏换焦点与依据  
6. 对 Claim **确认 / 拒绝** → 刷新后仍在

## 7. 评委一问一答备忘

**Q：和 ChatGPT / 知识库有什么不同？**  
A：持续项目态、本地授权、候选与已确认分离、判断回链到真实文件版本，不是一次性总结。

**Q：Agent 会不会乱写真相？**  
A：不会自动写成项目事实；确认前都是候选；失败不静默降级成假理解。

**Q：三帧 UI 全做完了吗？**  
A：故事与闭环已证明；候选节点版式与路径足迹是下一阶段 UI，当前用简报+画布材料链等价演示。
