## 3. 本产品交互硬规则（验收语言）

### 3.1 一屏主任务

| 允许 | 禁止 |
|------|------|
| 一句话现状（中文） | 英文模板空话当主文案 |
| 最多 3 个未知/冲突，人话 | 六问竖条问卷感 |
| 1 个主操作（如「让 Agent 再读一遍」） | 同时塞满 运行重建 / 保存 path / 停 watch / 画布 / 六问 |
| 确认 / 改一下 / 不行 | UUID、`candidate`、`revision`、`matter:`、HTTP adapter |

### 3.2 词表（Owner 可见层）

| 工程词 | 屏上说 |
|--------|--------|
| candidate | 待你确认 |
| revision | 一版理解 / 更新理解 |
| matter | 这件事 |
| evidence / path prefix | 依据 / 要看的文件夹范围 |
| reconcile | 对账中 / 已对齐 |

内部 ID **默认隐藏**；调试模式另开，不进默认 Owner 视图。

### 3.3 布局（先结构后皮）

```
┌─────────────┬──────────────────────────┬──────────────┐
│ 项目（窄）   │  现在怎样 + 依据（主）      │  你的决定     │
│ 一件事列表   │  变了什么 / 不清楚什么      │  确认·改·拒  │
└─────────────┴──────────────────────────┴──────────────┘
```

- **非线性画布**降为「需要时展开」，不是进门默认主舞台。  
- **来源授权 / path prefixes** 收进「设置范围」二级，不进首屏腰部。  
- **六问**改为后台结构；前台最多「现状 · 变化 · 不清楚 · 下一步」。

### 3.4 动效（克制版，先原则）

向 Rauno / Emil 学的是 **反馈清楚、短、有物理感**，不是全站动画秀。

| 场景 | 意图 | 约束 |
|------|------|------|
| 授权成功 / 对账完成 | 轻确认 | <300ms · reduced-motion 只淡入 |
| Agent 在读 | 进行中 | 可打断 · 不转圈骗完成 |
| 出候选 | 请你看 | 主卡片入场一次 · 不闪整页 |
| 确认/拒绝 | 结果落定 | 成功轻弹 · 错误不弹庆祝 |

完整 **Motion Recipe Protocol（A）** = 中长期 Agent 基建，**不阻塞**本轮信息架构改版。

## 4. 与「协议 A」的关系

| 层 | 是什么 | 何时 |
|----|--------|------|
| **现在** | 信息架构 + 文案 + 主路径按钮（上表） | 本波 |
| **A（动效配方）** | recipe YAML + SKILL + preview + 验证 | 主路径可读之后 |
| **采集源** | X 上 design engineer 作品 → 拆 intent/约束 | 建 A 库时 |

**判断：** 先把「不丑、能读、能决定」做对；再把「懂笑哭」的动效压成 A 给 Agent 复用。  
顺序反了会做出「动画很顶、还是看不懂」的壳。
