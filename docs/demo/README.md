# Demo 答辩入口

本场演示内容：一次客户对齐 → 双方交付承诺单 → 客户确认 → 服务方交付 → 客户验收。

## 先看什么

- [DEMO_SCRIPT.md](./DEMO_SCRIPT.md)：180 秒操作与话术。
- [BUSINESS_MODEL.md](./BUSINESS_MODEL.md)：商业假设；99 元/月仍是待验证价格，不作为已验证结论。
- [FC_ARCHITECTURE.md](./FC_ARCHITECTURE.md)：技术延展材料，不进入主 Demo。

## 已交付

- Web 粘贴与确定性演示对话。
- 服务方审阅后生成并发送承诺单。
- 无登录 `/c/[token]` 客户确认、修改、验收、拒收。
- 服务方不能代点客户确认或验收。
- 同 cohort 的 7 日确认率、确认耗时、按期验收候选指标。
- 单测、Playwright 演示步骤、lint 与生产构建门禁。

## 仍待验证 / 未交付

- ICP 是否愿意持续使用、99 元/月点击/付费意愿。
- 多租户、真实登录、云端持久化与公开生产部署。
- 微信机器人、原生端、法律电子签名、CRM、日历聚合均不在本次范围。

## 现场兜底

- 网络或 LLM 不稳定：点「载入演示对话」，fixture 不依赖外网。
- 客户状态未自动更新：点「刷新客户状态」。
- 本地服务重启会清空内存承诺单，重新按 5 步演示走一遍。
- 不说“已接入微信”，不由服务方代客户操作，不把候选指标说成市场验证。
