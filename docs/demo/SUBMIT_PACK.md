## 3. 使用说明（给评委 / 主办方）

### 环境

- Node 20+ 建议
- `npm install`
- 可选：LLM Key（无 Key 时会议拆解走离线规则，仍可演示）

### 三步试用

1. 打开 `/track/knowledge`（或首页点「打开知识工作台」）
2. 搜索框默认或输入 **「检索 来源」**，点「检索」→ 看到带来源标签的卡片
3. 右侧行动区，点某条待办上的状态按钮（Todo→Doing→Confirm→Done）

可选：

- 手记：写一句保存，再用关键词搜回来
- 会议：粘贴一段对话，生成卡片或待办（无模型时标离线）

### 边界

- 数据在本地 `data/knowledge/*.json`，重启一般还在（勿删该目录）
- 不做微信私聊对接、不做客户改约定产品
- 不把「向量库 / MCP 协议」当卖点吹满

### 自测命令

```bash
npm run test:unit
PORT=3011 BASE_URL=http://127.0.0.1:3011 npm run test:e2e
```

## 5. 路演 ≤5 页说法（大纲）

可直接做成 5 页 PPT，每页一句中心。

### 第 1 页 · 问题

知识工作者：材料散在聊天 / 纪要 / 文档。
找不回；找回了没出处；有结论落不成可勾的下一步。

### 第 2 页 · 我们做什么

**检索（带来源）→ 知识卡片 → 待办状态**
页面：`/track/knowledge`
用户先是自己，再讲效率赛道里的一人公司 / 小团队知识岗。

### 第 3 页 · 现场 Demo

屏幕上只跑：搜 → 卡带来源 → 点状态。
一句差异：不是第二个 Notion，也不是微信 CRM。

### 第 4 页 · 怎么做的（技术，短）

- Next.js 页面 + `/api/knowledge/*`
- 卡片 / 待办 JSON 落盘
- 模型可选（理解、拆任务）；**写进库的卡片和状态算数**；人点状态
- 工具列表：`/api/knowledge/mcp`（有则演示，不当唯一卖点）

### 第 5 页 · 商业与下一步

- 现在：自己先用满「搜得到、有来源、能勾掉」
- 以后：接真实来源（会议纪要、文档），再谈付费
- 明确不做：微信读私聊、客户改约定当第二产品、空吹体量数字
