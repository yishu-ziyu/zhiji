# 演示 Demo 脚本 - 交付运营助手 · 效率 OPC

> **版本**: 2026-07-13（台风改期：7/18 交作品 · 7/19 路演）  
> **赛道**: 一人公司效率提升 Agent  
> **产品**: 钉交付的运营助手  
> **北极星**: 闭环率 = 客户已确认交付 / 本期新增承诺  
> **总时长**: **180s**

---

## 核心金句

> **一人公司最稀缺的不是灵感，是注意力。**  
> **我们不帮你写会议纪要，我们帮你把客户承诺钉到交付。**

收尾金句：

> **闭环率可见，说到做到 —— 这就是一人公司的交付运营助手。**

## 一、时间切片总览（180s）

| 时段 | 时长 | 内容 |
|---|---|---|
| **0:00–0:25** | 25s | 钩子 + OPC 痛点 + 产品一句话 |
| **0:25–1:20** | 55s | 金标准剧本 → 承诺审阅 → 采纳进看板 |
| **1:20–2:25** | 65s | 状态推进 → 已确认 → 闭环率上升（核心证据） |
| **2:25–2:50** | 25s | 商业锚点（99 元 vs 雇助理）+ 评测方式 |
| **2:50–3:00** | 10s | 金句收尾 + 开放 Q&A |

**叙事弧**: 注意力有限 → 承诺会丢 → Agent 捕获与盯梢 → 数字可评测 → 一人公司可规模化交付

## 二、逐秒走查

### 第一幕 · 0:00–0:25 · 开场

| 秒 | 动作 | 话术 |
|---|---|---|
| 0–5s | 打开首页 `http://localhost:3000`，停在「交付运营助手」卡片 | （静默 1–2 秒） |
| 5–15s | 看评委 | 「评委老师好。一人公司最怕什么？不是不会聊客户，是**聊完之后没人盯交付**。」 |
| 15–22s | 点进交付运营助手 | 「纪要不重要。重要的是：他说了什么 → 我答应了什么 → **有没有交付、客户有没有确认**。」 |
| 22–25s | 页面加载，指顶部「闭环率」 | 「我们用一个北极星管这件事：**闭环率**。」 |

### 第二幕 · 0:25–1:20 · 捕获承诺

| 秒 | 动作 | 话术 |
|---|---|---|
| 0:25–0:35 | 点 **「使用客户对话剧本」** | 「这是一段真实形态的客户微信对齐：要原型、要预览链接，还说了句含糊的『体验好一点』。」 |
| 0:35–0:55 | 指「承诺审阅」列表 | 「Agent 拆出硬承诺和待澄清。**散文总结不算成功** —— 必须是可执行项。」 |
| 0:55–1:10 | 点 **「采纳并生成任务」** | 「一键进交付看板：已捕获 → 进行中 → 已交付 → 已确认。」 |
| 1:10–1:20 | 滚到看板 | 「没有项目经理编制时，这个看板就是你的交付会计账本。」 |

### 第三幕 · 1:20–2:25 · 闭环数字（最重要）

| 秒 | 动作 | 话术 |
|---|---|---|
| 1:20–1:40 | 把「输出落地页改版原型」：捕获 → 进行中 → 已交付 | 「开发做完，标记已交付。」 |
| 1:40–2:00 | 再点到 **已确认** | 「演示里用『客户已确认』按钮。真上线可以接客户回执；今天证明的是**状态机 + 可算指标**。」 |
| 2:00–2:15 | 指顶部闭环率从 0% → 有百分比 | 「看，闭环率动了。这就是『说到做到』的评测，不是感觉效率高了。」 |
| 2:15–2:25 | 可选：指逾期/风险区 | 「含糊需求进风险，不假装是可交付任务 —— 漏项可见。」 |

### 第四幕 · 2:25–3:00 · 商业 + 收尾

| 秒 | 动作 | 话术 |
|---|---|---|
| 2:25–2:40 | 可回首页或停在指标条 | 「雇一个盯交付的助理，一年十几万；我们目标是 **99 元/月级** 的交付运营助手。」 |
| 2:40–2:50 | — | 「技术上 Web 工作台今天可演示；Agent 内核与 UI 壳解耦，后续可进小程序点火。」 |
| 2:50–3:00 | 停画面在闭环率 | 「**闭环率可见，说到做到。** 谢谢各位，欢迎提问。」 |

## 四、评委追问速答

| 问题 | 答 |
|---|---|
| 这不就是看板吗？ | 看板是壳；价值是**从客户对话结构化承诺 + 强制状态机 + 闭环率**，解决 OPC 注意力泄漏。 |
| 和会议纪要工具有何不同？ | 纪要优化「写清楚」；我们优化「**有没有交付确认**」。 |
| 闭环率怎么算？ | 本期已确认任务数 / 本期新增（采纳）承诺数。 |
| 客户确认怎么做真的？ | 黑客松用创始人代点确认；产品路径是链接确认 / IM 回执 webhook。 |
| 为什么不做电商？ | 本场无真实店铺数据；效率侧用自己的交付流更诚实。电商可后续接真实账号。 |
| 算 Agent 吗？ | 捕获、分类硬/软/澄清、风险提示、状态推进与指标 —— 不是单次 chat 生成。 |
