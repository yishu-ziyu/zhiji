# 180 秒 Demo：双方交付承诺单

## 开演前

1. `npm run dev`，打开 `/track/efficiency`。
2. 准备同一浏览器的第二个标签页；客户链接无需登录。
3. 确认「载入演示对话」可离线返回固定剧本。

## 台本

| 时间 | 操作 | 话术 |
|---|---|---|
| 0:00–0:20 | 首页进入「双向交付承诺」 | 「项目制一人公司最怕的不是没记笔记，而是双方以为自己说清楚了，交付时才发现标准不同。」 |
| 0:20–0:45 | 点「载入演示对话」 | 「我只粘贴一次客户聊天，不声称接入微信。AI 把硬承诺和模糊点拆开；缺日期、缺验收标准就明确显示未知，不替客户编。」 |
| 0:45–1:35 | 审阅后点「生成并发送给客户」，复制并打开客户链接 | 「服务方先审阅草稿；发送后才签发无登录客户链接。服务方页面没有『客户已确认』按钮，因为客户动作不能由我代点。」 |
| 1:35–2:05 | 客户页点「确认承诺」；回服务方页点「标记已交付」 | 「客户自己确认事实；服务方完成后只能标记已交付，接下来仍由客户决定是否验收。」 |
| 2:05–2:30 | 客户页点「确认验收」 | 「现在双方闭环：谁确认、谁交付、谁验收都有动作记录。」 |
| 2:30–2:50 | 回服务方页指候选指标 | 「这里不展示伪精确的本期比率，而是同一创建 cohort 的 7 日确认率、确认耗时和按期验收候选指标。」 |
| 2:50–3:00 | 停在双方闭环卡片 | 「别证明 AI 能做多少；证明一次重复交易少一次漏约、返工和扯皮。」 |

## 失败兜底

- LLM 不可用：使用「载入演示对话」，走确定性 fixture。
- 客户页未自动更新：点「刷新客户状态」或刷新客户标签页。
- 本地服务重启：承诺单内存状态会清空，重新走一次 2 分钟黄金路径。

## 禁止话术

- 不说“已经接入微信”。
- 不由服务方代点客户确认或验收。
- 不声称本页面具备法律电子签名效力。
- 不说飞书、Notion 或 CRM “不能做”；只说优化对象不同。
