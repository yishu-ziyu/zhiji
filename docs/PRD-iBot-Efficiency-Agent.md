# iBot - 效率型知识 + 协作 Agent 产品需求文档 (v0.2)

## 1. 产品定位

**我们要做的是一个效率 Agent**，专为一人公司和小团队知识工作者设计。

它帮助用户把碎片化的会议、沟通、任务、资料**自动变成结构化、可检索、可行动的知识资产**。

## 2. 目标用户
- 首要用户：你自己（知识工作者）
- 次要用户：一人公司创业者、小团队项目负责人、运营/商务人员

## 3. 核心痛点 (具体、可验证)

1. 多走一步：每次会议后必须手动复制粘贴到 Notion/Excel
2. 多花时间：每周 2-4 小时在不同工具搜索历史信息
3. 多犯错误：遗漏行动项、版本混乱、跟进遗忘、知识断层

## 4. 核心能力 (精确定义)

### 4.1 自动会议纪要
**定义**：把原始会议输入（文字/语音转录）转化为结构化、可行动、可检索的输出。

**输出必须包含的字段**：
- 会议标题、时间、参与人
- 关键决策（带出处）
- 行动项列表（责任人 + 任务描述 + 截止时间 + 验收标准）
- 关联知识链接

**流程**：接收输入 → 提取事实 → 识别行动项 → 结构化 → 存知识库 → 推送确认

### 4.2 任务拆解
**定义**：把模糊目标拆成可验收的最小行动单元。

**每个任务必须包含**：任务描述、验收标准、截止时间、负责人、所需输入资料。

### 4.3 知识检索
**定义**：在个人知识库中找到最相关、可直接使用的片段。

**检索方案**：混合检索 (关键词 + 语义向量 + 时间衰减)

**检索后动作**：把结果作为上下文生成行动建议，同时记录本次检索行为。

### 4.4 协作状态更新
**定义**：自动推进任务状态并记录日志。

**更新后动作**：生成下一行动建议、通知相关人、沉淀经验到知识库。

### 4.5 行动建议
**定义**：基于知识检索 + 任务状态，给出具体、可执行的下一步行动（带理由）。

**可信机制**：每条建议都附带来源知识卡片链接，用户可一键验证。

## 5. 产品形态
Hybrid：独立 Web 应用 + MCP 可插拔能力
