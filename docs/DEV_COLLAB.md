# 开发协作

**用途：** 你和 Agent 共用说法与顺序。
**原则：** 平常中文；短；步骤心里有数，不必每次念出来。

---

## 1. 角色

| 谁 | 负责 | 不负责 |
|----|------|--------|
| **你** | 定目标、取舍、拍板、真机手感、对外提交 | 被要求填无限表格才开工 |
| **Agent** | 澄清验收、实现、对照、简短说明、跑测试 | 替你发明产品目标；偷偷改已定边界 |
| **工具链** | 编译、测试、Git、浏览器 | 替双方做产品决策 |

**你定「做什么、怎样算完」；Agent 做「做完并证明」。**

---

## 2. 五步

| 步 | 名称 | 谁 | 产出 | 怎样算完 |
|----|------|-----|------|----------|
| 1 | **意图** | 你 | 目标、边界、成功样子 | Agent 能复述 |
| 2 | **规格** | Agent | 做/不做、怎样算完；歧义最多问一轮 | 有可勾选验收 |
| 3 | **实现** | Agent | 最小可运行改动 | 按规格落地 |
| 4 | **对照** | Agent | 做到 / 未做 / 偏差 | 逐条对照 |
| 5 | **说明** | Agent | 路径、风险、下一步 | 短 |

对应 `AGENTS.md`：intent → clarify → implement → compare → explain。不要发明第三套名字。

## 3. 术语

### 协作

| 说法 | 意思 |
|------|------|
| **意图** | 你想要的结果与边界 |
| **规格 / Spec** | 可验收的约定：做/不做/怎样算完 |
| **验收** | 可勾 true/false 的检查 |
| **不做** | 明确排除 |
| **对照** | 拿结果逐条比规格 |
| **演示步骤** | 固定几步怎么点 |
| **交件** | 交给主办方/评委的材料与链接 |
| **手测** | 人在浏览器里点 |
| **e2e** | 浏览器/API 自动走用户可见路径 |

### 产品对象

| 说法 | 意思 |
|------|------|
| **工作项** | 一件要推进完的事（状态+负责人+下一步+时间线） |
| **依据 / 卡** | 带来源的可引用结论 |
| **来源** | 会议/邮件/聊天/文档/手记 |
| **事件 / 时间线** | 写回工作项的历史 |
| **下一步** | 当前一句要做的动作 |
| **负责人** | 人的名字或 agent id |
| **状态** | 字典里的枚举 |
| **知识工作台** | `/track/knowledge` |
| **知识足迹** | 依据卡在库图上的触达点亮 |
| **关系 / 边** | 两张依据之间带类型、带来源句的连接 |
| **来源句** | 边上可指回的一句依据 |

详见：

- `docs/SPEC-work-item-shared-situation.md`
- `docs/SPEC-knowledge-footprint.md`
- `docs/SPEC-knowledge-relations.md`

## 4. 流程

```text
意图 → 规格 → 实现 → 对照 → 说明
（需要时：提交、推送、更新交件包）
```

### 必须停下来问你

- 改产品边界（做/不做变了）
- 删除用户已有重要文件/数据
- 推送/改远程（若本轮未授权）
- 密钥、登录态、第三方账号
- 两个实现方向成本差一个数量级且不可逆

### 怎样算开发完

1. 验收条目已对照
2. 约定的测试已跑（或明示跳过原因）
3. 你知道：改了什么、怎么打开看、还有什么风险

---

## 5. 沟通

1. 先结论：做成了没有 / 卡在哪。
2. 给路径、URL、命令。
3. 对照规格，不要只报「优化了」。
4. 你说「不对」→ 改行为并写入约定。
