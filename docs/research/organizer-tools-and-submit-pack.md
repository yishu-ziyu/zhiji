## 一、必须交的作品（提交项）

标准口径：**可展示、可解释、可复盘**。评审看 Demo + 场景 + 商业 + 表达。

| 提交项 | 要求 | 我们项目对应物 |
|--------|------|----------------|
| **Agent Demo 或工作流** | 能演示核心流程；网页/链接/工作流/插件/脚本/录屏均可 | `npm run dev` → `/track/efficiency` + `/c/[token]`；录屏兜底 |
| **路演 PPT** | **5 页以内**：问题场景、解决方案、Demo 路径、工具/技术路线、商业化可能 | `docs/demo/DEMO_SCRIPT.md` + 待压成 ≤5 页 PPT |
| **MCP 协议或 MCP 链接** | 给 **AI 技术评分** 与 **MiXer AI** 侧评估；**不强制交源码** | 建议链接：https://open.hirebox.cn/ 生成 MCP；本仓若无可先说明 API 面 |
| **场景说明** | 解决谁的什么问题、为何有价值 | CONTEXT + 客户变更产品地图 |
| **使用说明** | 输入/输出/步骤/边界；必要时测试账号 | 黄金路径：载入演示对话 → 发送 → 客户确认/变更 |
| **技术/工具路线** | 模型、平台、AI Coding、开源、工作流 | Next.js + LLM 代理 + in-memory 状态机；见下节披露草稿 |
| **AI 使用披露** | 用了哪些 AI 工具/模型/模板/开源/素材 | 必写（见下方模板） |
| **团队信息** | 分工、联系方式、是否愿意入驻/认证/共创 | 个人参赛：角色自填 |
