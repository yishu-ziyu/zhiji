# 调研怎样才算「够」

> 针对：现有调研偏二手 V1、不能支撑「大家时间花在哪」。  
> 原则：先服务**我们自己想清楚**，比赛只是外生截止日，不降低证据标准的定义。

## 1. 现有调研为什么不够

| 已有 | 缺的 |
|------|------|
| 海外自雇/自由职业问卷（沟通压力、brief 不清） | **中国**、**接单变现**、**你这种内容→询盘**场景 |
| 竞品功能表（妙记、会议 AI） | 用户**一周时间账** |
| 产品假说与招募条件 | **真人说「痛」的录音/聊天** + 可复现工件 |
| 机制故事（两套事实） | **次数、时长、金钱损失** 任一项量化 |
| V0/V1 分级很诚实 | **V2**（可观察行为）**V3**（付费/留存）几乎为零 |

「够」不是再多几篇网文，而是：**能回答主办方那句反问的操作版**——  
**谁、在什么时刻、被什么卡住、一次卡多久、现在怎么凑合、愿不愿意为哪种解法花时间或钱。**

## 2. 够用的证据阶梯（可执行）

用同一套梯子，避免自我感动。

| 等级 | 名称 | 最低门槛 | 能支持的决策 |
|------|------|----------|--------------|
| **L0** | 自述 | 你自己讲感觉 | 只能开脑暴，不能定方向 |
| **L1** | 工件 | 脱敏聊天/订单/日历截图，能指到时间点 | 证明「发生过」，可画环 |
| **L2** | 结构访谈 | ≥5 个目标人，同一提纲，记「时间花在哪」 | 可排序痛点，仍弱于定量 |
| **L3** | 时间账 | ≥3 人 × ≥3 个工作日，或你自己 5 天日记 | 能回答「省谁的时间」 |
| **L4** | 行为验证 | 手工流程或原型介入真实单，前后对比 | 能说「这环该产品化」 |
| **L5** | 意愿/付费 | 预售、定金、持续使用 | 商业化叙事才硬 |

**在「不能访谈、时间紧」前提下的底线：**

- 主路径 = **高质量二手包**（§4.1 最低合格包）  
- 作者自述/一单时间线 = **可选插图**，不挡前进  
- L2 冷访谈 = **本阶段不做**  
- 二手再好也到不了 L4/L5；产品决策用「有依据的假说」，不用「市场已验证」话术
## 3. 必须填的一张表（调研产出物）

每个人 / 每单只填这种，不写长报告也行：

| 字段 | 说明 |
|------|------|
| 角色 | 如：独立开发接单 / 内容变现接单 / … |
| 一单从接触到收尾的阶段 | 用你的环命名，可改 |
| 每段估计耗时 | 分钟或小时；不确定就写区间 |
| 最烦的 1～2 段 | 为什么烦（找记录 / 重讲 / 白做 / 催款…） |
| 现用凑合办法 | 微信搜索、备忘录、表格、脑子 |
| 最近一次具体事故 | 何时、卡多久、损失（钱/关系/睡眠） |
| 若少一半时间愿意怎样 | 学工具 / 付费 / 不改 |

**合成指标（有 5 张表就能算）：**

- 沟通澄清占比 = 澄清耗时 / 全单耗时  
- 变更事故率 = 有「两套事实」的单 / 总单  
- 可产品化时间 = 烦且重复、且不是「真正做活」的时间  

## 4. 样本从哪来——修正（2026-07-14 用户反驳后）

**用户约束（必须尊重）：**

- 不认识足够多的一人公司/接单者，**访谈路径现实不可行**
- 截止日前**没有时间**做一手招募访谈
- 因此：**现成调研（二手、公开报告、平台数据、竞品公开材料）是主路径，不是将就**

先前「先不追求全国代表样本 / 去约 5 人访谈」在**有人脉与时间**时成立；在你当前条件下会变成假动作。  
**改口：默认不做冷启动访谈；把「够」定义在二手研究的质量上。**

### 4.1 二手为主时，怎样才算「做得好」

不是「多贴几篇链接」，而是同一问题用**可复核的公开证据**叠满：

| 要答的问题 | 二手可以怎么取证 | 仍不能假装 |
|------------|------------------|------------|
| 谁在花时间 | 行业报告里的角色/工种、岗位 JD、平台劳动者调查 | 「就是中国 OPC 全体」 |
| 时间花在哪类事 | 报告里的 top 痛点占比、工具使用调研、客服/交付工时拆分 | 精确到你个人的小时数（除非加自述） |
| 卡点是否重复 | 多份报告同方向 + 法院/平台规则/论坛高频帖 | 单条营销文 |
| 有没有人愿意为效率付钱 | 竞品定价与融资、SaaS 渗透、付费工具榜 | 你的产品已被验证愿付 |

**最低合格包（无访谈也能交卷给自己的思考）：**

1. **问题卡片 × N**：每张只问一件事（如「交付中途改需求」），下面挂 ≥2 个独立来源  
2. **来源表**：标题、地区/样本、年份、方法（问卷/面板/平台数据）、可信度 A/B/C、能支持的句子 / 不能支持的句子  
3. **时间叙事**：用报告数字拼出「典型一周/一单」的**示意账**（标明是合成示意，不是实测）  
4. **反证栏**：至少 1 条反对「人人都痛 / 人人会用 AI」的来源  
5. **映射**：每条证据 → 环上的哪一节 → 是否值得做进产品  

有 1～5，二手调研就「好看且硬」；没有 2 和 4，就只是剪报。

### 4.2 可选增强（有则更好，无也不阻塞）

## 5. 怎样才算「可以进入做产品形状」（二手主路径版）

同时满足即可（**不要求**完成 5 人访谈）：

1. **问题卡片**：环上候选 2～4 个卡点，各有来源表（§4.1）  
2. **合成时间叙事**：能讲清「这类人的时间大体耗在哪」，并标注数字出处与局限  
3. **排序理由**：为何先做某一环（证据强度 × 你的逻辑链 × 可实现），不是只因代码已有  
4. **反证**：至少写清「谁不痛 / 什么凑合办法已经够」  
5. **诚实边界**：哪些结论是海外/广义 SMB 外推，哪些不能写成「中国 OPC 已验证」

可选：作者自身一单作为插图，不升格为市场证明。

到这一步，「环环相扣」有公开证据托底；Agent 名词仍可后钉。
## 6. 和比赛截止日的关系（不畏手畏脚，但诚实）

- **证据标准**不因 17 号降低。  
- **决策勇气**可以：在 L1+L3 未满时仍可做原型，但嘴上必须说「假说」，不能说「调研证明」。  
- 若时间不够 L2，就公开：**以作者自身接单时间账为单案例研究**——这比假装有市场调研体面。

## 7. 下一手调研动作（二手主路径）

1. **盘点已有二手**：决策稿、竞品稿、Leapers/Malt/SMB 等 → 拆成问题卡片 + 来源表（能支持 / 不能支持）。  
2. **补洞检索**：专攻「时间花在哪」——交付变更、客户沟通工时、自由职业/个体户工具使用（中英文公开报告、平台年报、认真写方法的调研）。  
3. **合成一页「示意时间账」**：环 × 耗时占比（全部带来源脚注）。  
4. **访谈不排期**；若你偶然遇到合适的人再记，不作关卡。
